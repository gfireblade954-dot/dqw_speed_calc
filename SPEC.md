# DQW素早さ計算アプリ 仕様書

## アプリの目的
A→B→C→Dの行動順を確定させるための
素早さを逆算・確認するツール

## 用語定義
- 行動値 = 素早さ × N ÷ 100（切り捨て）
- 通常バトル: N = 100〜115（16通り）
- メガ/ギガ/魔王地図: N = 100〜120（21通り）
- 同値の場合はA→B→C→D（左）優先
- 先攻率 = 自分が先攻になるNの組み合わせ数 ÷ 全組み合わせ数 × 100

## 画面構成

### 上部：バトル種別切り替え
- 「通常（×1.15）」「メガ/ギガ/魔王（×1.20）」のセグメントボタン
- 切り替えで全計算結果が即時更新

### メイン：キャラ入力エリア（縦並び4枚）
各カードの構成：
- キャラ名（Aさん／Bさん／Cさん／Dさん）
- 素早さ入力欄（未入力時は「素早さを入力」プレースホルダー）
- 結果表示エリア（下記ロジックで表示）

## 表示ロジック

## 比較ルール（最重要）
- 各キャラは「左隣のキャラ」とのみ比較する
- Aさんのみ例外で右隣のBさんと比較
- 入力済みキャラが複数いても比較対象は常に左隣1人のみ
- 全体の行動順リストは表示しない

### 入力済みキャラの表示
左隣（Aのみ右隣）のキャラも入力済みの場合：
- 自分が上回っている → 「XX.XX%で先攻」
- 自分が下回っている → 「XXX以下なら後攻」
- 確定先攻の場合 → 「確定先攻」
- 確定後攻の場合 → 「確定後攻」

自分が入力済みで比較対象が未入力の場合：
- 「確定必要値：XXX」のみ表示
  （確定必要値 = 比較対象のactMax + 1 になる自分の素早さ）

### 未入力キャラの表示
左隣（Aのみ右隣）が入力済みの場合：
- 「XXX以下で後攻」を表示
  （XXX = 左隣のactMin - 1 になる素早さ）
左隣も未入力の場合：
- 何も表示しない

## 先攻率の計算
通常: 16×16=256通り、メガ: 21×21=441通りを全列挙
- 自分のN × 相手のN の全組み合わせを計算
- 自分の行動値 > 相手の行動値 → 先攻カウント
- 自分の行動値 = 相手の行動値 → 左側（index小）が先攻
- 先攻率 = 先攻カウント ÷ 全組み合わせ数 × 100（小数点2桁）

## 確定必要値
- 相手のactMax(素早さ × nMax ÷ 100 切り捨て) + 1

## エッジケース
- 素早さは1以上の整数のみ受付
- 0・負の値・文字は入力不可
- 全員未入力時は結果エリアを空表示

## やらないこと
- モンスター側との比較
- データ保存・履歴
- キャラ名のカスタマイズ
- 5人以上の比較
## レイアウト
- スマホ（幅600px未満）: A→B→C→D を縦1列に並べる
- PC・タブレット（幅600px以上）: A→B→C→D を横1列に並べる
- FlutterはLayoutBuilderで画面幅を取得して切り替える
- 各カードの高さは固定せず内容に合わせて可変

## 未入力キャラの連鎖計算ルール

### 考え方
未入力キャラは「左隣が後攻になる素早さの上限値」を
自分の素早さと仮定して連鎖的に計算する。

### 計算手順
1. 左側で最初に入力済みのキャラを起点にする
2. 起点の右隣（未入力）の後攻上限値を計算する
   後攻上限値 = floor(起点のactMin - 1) を素早さに逆算
   つまり = 起点素早さのactMin - 1
   （actMin = 素早さ × 100 ÷ 100 = 素早さそのまま）
   よって後攻上限値 = 起点素早さ - 1
3. その後攻上限値を次のキャラの仮素早さとして
   さらに右隣の後攻上限値を連鎖計算する

### 具体例（通常バトル、A=1000の場合）
- Bの後攻上限 = floor(A×100÷100) - 1 = 1000 - 1 = 999？

※ 注意：実際はactMin(1000)=1000なので
   Bが後攻になるにはB×115÷100 < 1000
   つまりB < 870 → B=869以下で後攻

- C後攻上限 = Bの869を仮素早さとして同様に計算
- D後攻上限 = Cの結果を仮素早さとして同様に計算

### 実装メモ
```dart
// 後攻上限素早さの逆算
// 相手のactMin > 自分のactMax が後攻確定条件
// 自分のactMax < 相手のactMin
// 自分 × nMax ÷ 100 < 相手素早さ
// 自分 < 相手素早さ × 100 ÷ nMax
int behindUpperLimit(int leftSpd) {
  return (leftSpd * 100 / nMax).ceil() - 1;
}
```

### 入力済みキャラが途中にいる場合
- 入力済みキャラは実際の入力値を使う
- 未入力キャラのみ連鎖計算値を使う
- 例：A=1000（入力）、B未入力、C=800（入力）、D未入力
  → B：869以下で後攻（Aから連鎖）
  → C：800入力済みなので実値使用
  → D：Cの800から逆算して後攻上限を表示
